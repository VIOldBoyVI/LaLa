# LaLaGame - Полное описание проекта

## Общая информация

**Название проекта:** ЛА-ЛА-ГЕЙМ - Викторина с элементами караоке
**Тип проекта:** Веб-приложение-викторина
**Технологии:** Python, Flask, SQLite, HTML, CSS, JavaScript

## Описание игры

ЛА-ЛА-ГЕЙМ — веб-игра-викторина с элементами караоке. Игра выполнена в виде веб-приложения с интерактивным полем 10×10, где игроки открывают ячейки, отвечают на вопросы и зарабатывают очки.

### Особенности игры:
- Игровое поле 10×10 с буквенной и числовой адресацией
- 5 раундов: 4 раунда по 20 вопросов и 1 финальный с 10 вопросов
- Перерывы после 2-го и 4-го раундов (по 10 минут)
- Каждый раунд имеет свою тему (музыкальные жанры, исполнители, инструменты и т.д.)
- Система подсчёта очков
- Сохранение состояния игры между сессиями
- Адаптивный дизайн для настольных компьютеров

### Backend (Python Flask):
- **Фреймворк:** Flask 2.3.3
- **Класс приложения:** QuizGameApp
- **База данных:** SQLite с тремя таблицами:
  1. `questions` - вопросы для всех раундов (всего 720 вопросов)
  2. `game_states` - состояние текущих игр
  3. `scores` - очки игроков

### Frontend:
- **HTML шаблоны:** 3 различных версии с разными цветовыми схемами
- **CSS:** Адаптивный дизайн с градиентами и анимациями
- **JavaScript:** Логика игры, обработка кликов, подсчет очков

## Игровые раунды

### Темы раундов:
1. **Раунд 1:** Музыкальные жанры (160 вопросов)
2. **Раунд 2:** Музыкальные исполнители (160 вопросов)
3. **Раунд 3:** Музыкальные инструменты (160 вопросов)
4. **Раунд 4:** Музыкальная история (160 вопросов)
5. **Раунд 5:** Смешанные темы (80 вопросов)

## Технические особенности

### Безопасность:
- Использование SECRET_KEY для Flask-приложения
- Защита от SQL-инъекций через параметризованные запросы
- Поддержка CORS для кросс-доменных запросов

### Архитектура кода:
- Организация кода в классы для лучшей структуры
- Аннотация типов для лучшей читаемости
- Четкая документация функций и классов
- Улучшенная обработка ошибок

### API эндпоинты:
- `POST /api/init_game` - инициализация новой игры
- `POST /api/get_question` - получение вопроса для раунда
- `POST /api/check_answer` - проверка ответа на вопрос
- `POST /api/save_state` - сохранение состояния игры
- `GET /api/load_state` - загрузка состояния игры

## Игровой процесс

1. Игроки открывают ячейки на игровом поле (10×10)
2. При открытии ячейки с номером появляется вопрос
3. При правильном ответе игрок получает 10 очков
4. После 20 открытых ячеек в раундах 2 и 4 начинается 10-минутный перерыв
5. Всего 5 раундов, после которых игра завершается

## Игровое поле

- 100 ячеек (10×10)
- 90 ячеек с вопросами (числа 1-90)
- 10 ячеек с символами (★, ♥, ♦, ♣, ♠, ♪, ♫, ☀, ☁, ☂)
- Буквенная адресация столбцов (A-J)
- Числовая адресация строк (1-10)
